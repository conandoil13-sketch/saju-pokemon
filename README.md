# 생년월일시로 보는 나의 포켓몬

생년월일시 기반의 간지/오행 구조를 가볍게 해석해 포켓몬 타입으로 바꿔보고, 그 타입과 어울리는 포켓몬을 찾아보는 팬메이드 웹 미니게임입니다.

이 프로젝트는 정확한 만세력 앱이나 운세 풀이 서비스가 아닙니다. 오행 분포를 포켓몬 타입에 연결해보는 타입 매칭 놀이입니다.

## 팬메이드 고지

본 사이트는 비공식 팬메이드 프로젝트이며, 공식 포켓몬 콘텐츠가 아닙니다.

Pokémon, 포켓몬 이름, 타입 등 관련 권리는 Nintendo / Creatures / GAME FREAK / The Pokémon Company에 있습니다.

## MVP 범위

- 초기 MVP는 양력 생년월일시 입력만 지원합니다.
- 음력 변환은 구현하지 않습니다.
- 생시를 모르는 사용자를 위해 `시간 모름` 옵션을 제공합니다.
- 기존 MVP 근사 계산에서 정확도 개선 핫픽스를 진행했습니다.
- 월주는 양력 월 고정값이 아니라 주요 절기 절입일 기준 근사값을 사용합니다.
- 일주는 검증 기준일 기반 60갑자 순환으로 계산합니다.
- 결과 카드에는 직접 추가한 이미지 또는 PokeAPI 이미지 URL을 표시합니다.
- PNG 저장용 결과 카드는 모바일 공유에 맞춘 세로형 비율로 렌더링합니다.

## 사주 계산 기준

이 프로젝트의 목적은 정확한 만세력 제공이 아니라 오행 기반 포켓몬 타입 매칭 놀이입니다.

- 현재 계산 모드는 `improved-local`입니다. 별도 라이브러리나 빌드 도구는 추가하지 않았습니다.
- 실제 사주 계산에서는 절기 기준 월주, 입춘 기준 년주, 정확한 일주, 일간 기반 시주가 중요합니다.
- 년주는 입춘 날짜 기준으로 전년도/당해년을 나누는 구조를 반영했습니다.
- 월주는 실제 양력 월이 아니라 주요 절기 절입일 기준으로 월지를 고릅니다. 다만 절입 시각까지 계산하지는 않으므로 전문 만세력과 하루 정도 차이가 날 수 있습니다.
- 일주는 `2000-01-07 = 갑자일` 기준 60갑자 순환으로 계산합니다.
- 시주는 일간별 자시 천간 시작 규칙을 사용합니다.
- 생시를 모르면 시주는 계산과 오행 집계에서 제외합니다.
- 오행 카운트는 기본적으로 천간/지지 겉글자 8글자 기준입니다.
- 지장간, 계절 가중치, 월령 가중치, 신강신약 판단은 아직 반영하지 않았습니다.
- 외부 만세력 앱과 비교 검증을 위해 오행 직접 보정 모드를 제공합니다.

## 데이터와 API

기본 배포에서는 커밋된 정적 도감 파일 `data/pokemon-pool.js`를 먼저 사용합니다.
이렇게 하면 GitHub Pages 방문자가 매번 PokeAPI에 1025마리 데이터를 요청하지 않습니다.

- `pokemon/{id}` 엔드포인트에서 도감번호, 영어 이름, 타입 정보를 가져옵니다.
- `pokemon-species/{id}` 엔드포인트에서 `names` 배열을 가져오고, `language.name`이 `ko`인 항목을 한글명으로 사용합니다.
- 한글명이 없으면 영어명을 fallback으로 사용합니다.
- 포켓몬 설명, 능력치, 기술 정보는 가져오지 않습니다.
- 이미지 URL은 PokeAPI의 `sprites.other["official-artwork"].front_default`를 우선 사용하고, 없으면 기본 sprite로 fallback합니다.
- 직접 제작한 이미지를 쓰려면 `assets/images/pokemon/0006.png`처럼 4자리 도감번호 파일명으로 넣습니다.
- 직접 제작한 이미지가 있으면 그 이미지를 먼저 쓰고, 없으면 PokeAPI 이미지 URL을 사용합니다.
- 정적 도감 파일은 `node scripts/build-pokemon-pool.mjs`를 실행해 PokeAPI에서 한 번 수집한 뒤 갱신합니다.
- API 요청 실패에 대비해 최소 fallback 데이터를 둡니다.
- 정적 도감 파일이 비어 있거나 강제 갱신을 할 때만 브라우저 fetch와 `localStorage` 7일 캐시를 사용합니다.
- 네트워크 실패 또는 API 요청 실패 시 fallback 데이터를 사용합니다.
- 본 사이트는 팬메이드이며 공식 포켓몬 콘텐츠가 아닙니다.

## 공유와 PNG 저장

- 결과 텍스트 복사, Web Share API 기반 공유, PNG 저장 버튼을 제공합니다.
- PNG 저장은 결과 상세 전체가 아니라 공유용 세로 결과 카드 영역만 캡처합니다.
- 브라우저 보안 정책상 사진 앱에 직접 저장하지는 않고, 지원 브라우저에서는 공유/저장 시트를 열며 그 외에는 PNG 다운로드로 fallback합니다.
- PNG 캡처에는 CDN으로 로드한 `html2canvas`를 사용합니다.

## 오행 궁합 포켓몬

결과 카드에는 “나랑 잘 맞는 포켓몬”과 “나랑 안 맞는 포켓몬”이 함께 표시됩니다.

- 이 기능은 정밀 사주 궁합이 아니라 오행 상생/상극 관계를 바탕으로 만든 팬메이드 타입 매칭입니다.
- 잘 맞는 포켓몬은 사용자의 주 오행을 생해주는 오행 타입군에서 고릅니다.
- 부딪히기 쉬운 포켓몬은 사용자의 주 오행을 극하는 오행 타입군에서 고릅니다.
- 예를 들어 주 오행이 `火`이면 `木` 계열은 상생, `水` 계열은 상극으로 봅니다.
- 포켓몬 선택은 생년월일시 seed 기반으로 고정되므로 같은 입력값은 같은 궁합 포켓몬을 보여줍니다.
- 실제 인간관계나 운세 판단이 아니라 공유용 미니게임 결과입니다.

## 실행 방법

빌드 도구 없이 정적 파일만으로 동작하도록 구성합니다.

1. 저장소를 내려받습니다.
2. GitHub Pages에 그대로 배포할 수 있습니다.
3. 로컬에서 ES module import까지 확인하려면 정적 서버로 열어보는 것을 권장합니다.
